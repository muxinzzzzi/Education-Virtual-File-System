## 🔧 实现方案

### 需要添加的新协议命令

1. `LIST_MY_PAPERS` - 作者获取自己的论文列表
2. `LIST_ASSIGNED_PAPERS` - 审稿人获取分配给自己的论文
3. `LIST_ALL_PAPERS` - 编辑获取所有论文（支持分页）

### 客户端改进

1. **表格渲染工具**
   - 自动计算列宽
   - 支持多行内容截断
   - 美观的边框

2. **分页逻辑**
   - 每页5条记录
   - 页码显示
   - 上一页/下一页/首页/尾页

3. **子菜单系统**
   - 列表视图 → 详情视图
   - 面包屑导航
   - 返回上级菜单

## 🎬 用户体验流程

### 作者流程
```
主菜单 → [2] 查看论文状态
  ↓
显示论文列表（表格）
  ↓
选择论文 [1-N] → 显示详细信息
  ↓
[0] 返回列表 → 回到列表
  ↓
[0] 返回主菜单 → 回到主菜单
```

### 审稿人流程
```
主菜单 → [3] 查看审稿状态
  ↓
显示待审论文列表
  ↓
选择论文 → 显示详情 + 快捷操作（下载/提交审稿）
  ↓
返回列表
```

### 编辑流程
```
主菜单 → [4] 查看审稿进度
  ↓
显示论文列表（第1页，共N页）
  ↓
[N]下一页 / [P]上一页
  ↓
选择论文 → 详细审稿进度
  ↓
返回列表
```

## 📝 注意事项

1. **性能考虑**
   - 论文数量可能很多，需要分页
   - 服务器端需要支持分页查询

2. **用户体验**
   - 表格要对齐美观
   - 长文本要截断
   - 空列表要有友好提示

3. **一致性**
   - 所有角色使用相同的表格风格
   - 统一的快捷键（0返回，N下一页，P上一页）
   - 一致的颜色方案
